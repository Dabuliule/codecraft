# CodeCraft v0.1.0a3

[English](v0.1.0a3.md)

CodeCraft `v0.1.0a3` 是一个 Alpha 版本，重点是把项目完善为完整、可检查的
Coding Agent 应用。本版本加入全屏 TUI、仓库检索、评测与 trace 报告、MCP
互操作、分层沙箱和更严格的运行时治理。

## 主要更新

- **交互式应用：** 裸 `codecraft` 命令打开 Textual TUI，支持多轮对话、
  审批弹窗、session 恢复和 trace 查看。
- **仓库上下文：** 增量本地索引、符号与词法检索、确定性查询路由、写入后的
  自动刷新，以及模型可调用的 `workspace_search` 工具。
- **可量化的 Agent 质量：** 固定的 10 题 Agent 评测集和独立检索基准均可生成
  可重复的 JSON/HTML 报告。
- **互操作：** 配置的 MCP server 会成为受运行时治理的 Agent 工具；CodeCraft
  也可以通过自身 MCP server 暴露只读仓库上下文。
- **分层执行安全：** 默认使用 macOS Seatbelt 和 Linux bubblewrap 进行原生隔离；
  Docker 仍可用于可复现、带资源限制的执行环境。
- **运行时韧性：** 严格的模型与工具协议、分层超时、上下文压缩、受限的只读
  并发、输出限制、阶段耗时和可恢复事件日志。

## 不兼容变更

- 裸 `codecraft` 命令现在启动全屏 TUI，替代上一版的行式交互界面。
- `v0.1.0a2` 中的 `codecraft chat` 和 `codecraft resume` 子命令已移除。
  多轮工作改用 TUI，自动化与诊断继续使用具名 CLI 命令。
- CodeCraft 仍是 Alpha 软件，不保证与更早 Alpha 版本中持久化的 session 和
  配置 schema 兼容。

## 安装或升级

需要 Python 3.11 或更高版本。使用 `uv` 安装这一确切版本：

```zsh
uv tool install --force \
  "git+https://github.com/Dabuliule/codecraft.git@v0.1.0a3"
codecraft
```

GitHub Release 同时提供通用 wheel 和 source distribution。本版本未发布到 PyPI。

## 验证

- CI 在 Python 3.11 和 3.12 上通过。
- 非 opt-in 自动化测试结果为 `165 passed, 2 skipped`。
- Docker 沙箱集成测试在 GitHub Actions 中运行。
- Ruff 格式和 lint 检查通过。
- Wheel 与 source distribution 元数据通过 Twine 校验。
- 首次真实 provider 评测基线记录在
  [docs/EVAL_BASELINE.md](https://github.com/Dabuliule/codecraft/blob/v0.1.0a3/docs/EVAL_BASELINE.md)。

## 已知限制

- 仓库检索目前是本地确定性实现，本版本不包含基于 embedding 的语义检索。
- 配置的 stdio MCP server 进程在宿主机启动。其发现的工具调用仍会经过正常的
  沙箱、审批和 trace 治理，但尚未自动对 server 进程本身进行 Docker 隔离。
- 真实 provider 评测基线目前只有一次完整的 10 题运行；在把总体分数视为稳定
  指标前，还需要更多重复运行。
- 应用设计仍在完善，Alpha 阶段的兼容范围有意保持较窄。
