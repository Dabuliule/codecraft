# CodeCraft v0.1.0a3

[简体中文](https://github.com/Dabuliule/codecraft/blob/main/docs/releases/v0.1.0a3.zh-CN.md)

CodeCraft `v0.1.0a3` is an alpha release focused on turning the project into a
complete, inspectable coding-agent application. It adds a full-screen TUI,
repository retrieval, evaluation and trace reports, MCP interoperability,
layered sandboxing, and stricter runtime governance.

## Highlights

- **Interactive application:** bare `codecraft` opens the Textual TUI with
  multi-turn conversation, approval modals, session resume, and trace viewing.
- **Repository context:** incremental local indexing, symbol/lexical retrieval,
  deterministic query routing, automatic refresh after writes, and a model-facing
  `workspace_search` tool.
- **Measurable agent quality:** a fixed 10-task agent evaluation suite and a
  separate retrieval benchmark produce repeatable JSON/HTML reports.
- **Interoperability:** configured MCP servers become governed agent tools, while
  CodeCraft can expose read-only repository context through its own MCP server.
- **Layered execution safety:** native macOS Seatbelt and Linux bubblewrap are the
  default process isolation path; Docker remains available for reproducible,
  resource-limited execution.
- **Runtime resilience:** strict model/tool protocols, layered timeouts, context
  compaction, bounded read-only concurrency, output limits, phase timings, and
  recoverable event logs.

## Breaking Changes

- Bare `codecraft` now starts the full-screen TUI instead of the previous
  line-oriented interactive interface.
- The `codecraft chat` and `codecraft resume` subcommands from `v0.1.0a2` were
  removed. Use the TUI for multi-turn work and named CLI commands for automation
  and diagnostics.
- CodeCraft is still alpha software. Persisted session and configuration schemas
  from earlier alpha builds are not guaranteed to be compatible.

## Install Or Upgrade

Python 3.11 or newer is required. Install this exact release with `uv`:

```zsh
uv tool install --force \
  "git+https://github.com/Dabuliule/codecraft.git@v0.1.0a3"
codecraft
```

The GitHub release also includes a universal wheel and source distribution. This
release is not published to PyPI.

## Validation

- CI passes on Python 3.11 and 3.12.
- The automated suite reports `165 passed, 2 skipped` for non-opt-in tests.
- Docker sandbox integration runs in GitHub Actions.
- Ruff formatting and lint checks pass.
- Wheel and source-distribution metadata pass Twine validation.
- A first real-provider evaluation baseline is documented in
  [docs/EVAL_BASELINE.md](https://github.com/Dabuliule/codecraft/blob/v0.1.0a3/docs/EVAL_BASELINE.md).

## Known Limitations

- Repository retrieval is local and deterministic; embedding-based semantic
  retrieval is not included in this release.
- Configured stdio MCP server processes start on the host. Their discovered tool
  calls still pass through normal sandbox, approval, and trace governance, but
  automatic process-level Docker isolation for the server itself is not yet
  implemented.
- The real-provider evaluation baseline contains one full 10-task run. More
  repeated runs are needed before treating its aggregate score as stable.
- Alpha compatibility remains intentionally narrow while the application design
  is still being refined.
